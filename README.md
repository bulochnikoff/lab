# Лабораторная работа по Docker

## Описание сервисов

**db** – сервис базы данных на основе PostgreSQL 15. Отвечает за хранение данных. При первом запуске инициализируется скриптом `db/init.sql`, который создаёт таблицу `test` и заполняет её тестовой записью. Данные сохраняются в томе `pgdata`.

**backend** – сервис бэкенда на Flask. Предоставляет REST API, которое подключается к базе данных по имени `db` (благодаря общей сети). Возвращает сообщение из таблицы `test`. Проброшен порт `5000` для доступа извне.

**frontend** – сервис фронтенда на Nginx. Отдаёт статическую HTML-страницу, которая с помощью JavaScript (Fetch API) обращается к бэкенду по адресу `http://localhost:5000` и отображает полученное сообщение. Проброшен порт `8080` для доступа через браузер.

## Инструкцией по первому запуску.

Выполните команду `docker-compose up --build -d` для сборки образов и запуска контейнеров в фоновом режиме. Проверьте статус контейнеров командой `docker-compose ps` – все сервисы должны быть в состоянии Up. Откройте в браузере фронтенд по адресу `http://localhost:8080` и бэкенд напрямую по адресу `http://localhost:5000`. Для остановки контейнеров используйте docker-compose down, а для полной очистки с удалением тома базы данных – `docker-compose down -v`.

## Диаграмма архитектуры

┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│ frontend │ ──────▶ │ backend │ ──────▶ │ db │
│ (nginx) │ │ (flask) │ │ (postgres) │
└─────────────┘ └─────────────┘ └─────────────┘
   порт 8080      порт 5000          порт 5432
   │
   └─ app-network (общая сеть)
   
Все сервисы находятся в одной Docker-сети `app-network` и могут обращаться друг к другу по именам контейнеров (например, бэкенд подключается к базе по адресу `db`).
